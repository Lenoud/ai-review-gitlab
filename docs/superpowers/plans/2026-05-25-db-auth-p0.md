# DB + Auth P0 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the first usable database and authentication slice for the Gin backend.

**Architecture:** Keep the current layered shape. `cmd/server` wires config, GORM database, repositories, services, handlers, and router dependencies. Production uses real JWT middleware; router tests can inject a test auth middleware.

**Tech Stack:** Gin, GORM, MySQL driver, bcrypt, golang-jwt/jwt/v5, testify.
